Contributing

Feel free to fork the Project.
Contributions are welcome in the following areas:

  • docs/
  • stdlib/

You can get swag such as stickers. Just ask.

Issues, Bugs Reports, Suggestions are also welcome.

If you have a Project using it, send an email to get it featured :)

Contributing to docs/

  • Docs should stay static html+css only, with no external dependency other than Bootstrap
  • There should only ever be one custom css file, namely css/style.css
  • Target Devices are Firefox on Desktop and Mobile, but there should be no browser-specific css.
  • Docs should use the full width in Mobile Browsers
  • Everything outside the outermost .container div should have the same structure in all html files.
  • Example Workflow to add a new page: 
    cp html/skeleton.html html/my-file.html
vim html/my-file.html
vim index.html
  • What needs to be edited from the html/skeleton.html?
    • title
    • meta name description
    • meta name author
    • h1 inside container
    • actual content

Contributing to stdlib/

There are multiple ways to contribute to stdlib:

  • implementation in stdlib/
  • documentation of stdlib/

Currently there is a need for well tested implementations of

  • json reader/writer
  • generic collections
  • generic optional type
