SmallDragon Primitive Data Types

The primitive Data Types available in SmallDragon are used similar to those available in C, and are a subset of those.

Writing the types like 'int' / 'int32' / 'uint' / ...
was chosen because, opposed to 'i32' / 'u8' like in Rust, it could be more familiar to people knowing C / C++ / Java.
It omits the '_t' at the end of the fixed-size types in C, because these '_t' are IMO simply unneccessary characters added to the type.

Signed Integer Types

Type Width transpiles to
int8 1 byte int8_t
int16 2 bytes int16_t
int32 4 bytes int32_t
int64 8 bytes int64_t
int int (for C compatibility reasons)

Unsigned Integer Types

Type Width transpiles to
uint8 1 byte uint8_t
uint16 2 bytes uint16_t
uint32 4 bytes uint32_t
uint64 8 bytes uint64_t
uint 4 bytes uint32

float f = 3.14159;
